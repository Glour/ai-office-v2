# Writing Plans - Пошаговые планы реализации

## 📦 Установка

1. Скопируйте папку `writing-plans/` в `skills/` вашего workspace
2. OpenClaw автоматически подхватит скилл при следующем сообщении
3. Проверка: скажите агенту "составь план как сделать..."

## 🎯 Что умеет

- Превращать идеи и спецификации в пошаговые планы
- Разбивать задачи на атомарные шаги (2-10 минут каждый)
- Добавлять проверку выполнения для каждого шага
- Масштабировать детализацию под тип задачи
- Сохранять планы для будущего выполнения

## ⚙️ Как работает

### Главный принцип

**Пиши план так, будто его будет выполнять будущий-ты после рестарта - без контекста текущей сессии.**

Каждый шаг должен быть самодостаточным. Никаких "как обсуждали ранее" или "тот файл".

### Структура плана

#### Шапка (обязательно)

```markdown
# [Название] - План реализации

**Цель:** Одно предложение - что получим в итоге
**Срок:** Оценка времени
**Зависимости:** Что нужно до начала (инструменты, доступы, данные)

---
```

#### Задачи

Каждая задача - один атомарный шаг (2-10 минут):

```markdown
### Шаг N: [Действие]

**Что делаем:** Конкретное описание
**Файлы:** Точные пути (создать/изменить/удалить)
**Команды:** Точные команды если есть
**Проверка:** Как убедиться что шаг выполнен
**Результат:** Что получаем после этого шага
```

### Гранулярность по типу задачи

**Техническая задача (скрипт, автоматизация):**
- Шаг = одно действие: написать, запустить, проверить, закоммитить
- Точные пути к файлам
- Точные команды
- Ожидаемый вывод

**Контентная задача (пост, видео):**
- Шаг = один этап: исследование, драфт, правка, публикация
- Ссылки на скилы (copywriter, youtube-seo)
- Референсы и источники

**Организационная задача (переезд, документы):**
- Шаг = одно действие: позвонить, заполнить, отнести
- Контакты и адреса
- Дедлайны

### Правила написания планов

**Конкретные пути**
```
❌ "файл конфига"
✅ "~/.openclaw/openclaw.json"
```

**Конкретные команды**
```
❌ "запусти тест"
✅ "python3 scripts/test.py --verbose"
```

**Проверка каждого шага**
```
❌ "Создать файл"
✅ "Создать файл → Проверка: ls -la file.md показывает файл"
```

**Нет "потом разберёмся"**
Каждый шаг должен быть завершён и проверен до перехода к следующему.

**YAGNI (You Aren't Gonna Need It)**
Только необходимые шаги. Без "на всякий случай добавим".

**DRY (Don't Repeat Yourself)**
Если шаг повторяется - вынести в скрипт или функцию.

### Куда сохранять план

- **Технический** → `memory/plans/YYYY-MM-DD-название.md`
- **Контентный** → в daily notes
- **Разовый простой** → прямо в чат, без файла

### После создания плана

Агент предложит выбор:
1. **Сделать сейчас** - пошагово в этой сессии
2. **Субагент** - запустить автономное выполнение
3. **Крон** - если задача по расписанию
4. **Сохранить на потом** - записать и вернуться

## 💡 Примеры использования

**Пример 1: Технический план (скрипт)**

```markdown
# Скрипт бэкапа базы данных - План реализации

**Цель:** Автоматический бэкап SQLite базы каждую ночь
**Срок:** 1 час
**Зависимости:** sqlite3, доступ к базе, папка для бэкапов

---

### Шаг 1: Создать папку для бэкапов

**Что делаем:** Создаём директорию с правильными правами
**Команды:**
```bash
mkdir -p ~/backups/db
chmod 700 ~/backups/db
```
**Проверка:** `ls -la ~/backups/` показывает папку db
**Результат:** Папка готова для бэкапов

---

### Шаг 2: Написать скрипт бэкапа

**Что делаем:** Скрипт копирует базу с датой в имени
**Файлы:** Создать `scripts/backup-db.sh`
**Содержимое:**
```bash
#!/bin/bash
DATE=$(date +%Y-%m-%d_%H-%M)
DB_PATH="$HOME/.app/database.sqlite"
BACKUP_PATH="$HOME/backups/db/backup_$DATE.sqlite"

sqlite3 "$DB_PATH" ".backup '$BACKUP_PATH'"
echo "Backup created: $BACKUP_PATH"

# Удалить бэкапы старше 7 дней
find ~/backups/db -name "*.sqlite" -mtime +7 -delete
```
**Команды:** `chmod +x scripts/backup-db.sh`
**Проверка:** `bash scripts/backup-db.sh` создаёт файл в ~/backups/db/
**Результат:** Скрипт готов к использованию

---

### Шаг 3: Добавить в cron

**Что делаем:** Запуск каждую ночь в 3:00
**Команды:**
```bash
crontab -e
# Добавить строку:
0 3 * * * /path/to/scripts/backup-db.sh >> ~/backups/db/backup.log 2>&1
```
**Проверка:** `crontab -l` показывает задачу
**Результат:** Автоматический бэкап настроен

---

### Шаг 4: Тест и документация

**Что делаем:** Проверить что всё работает, записать в доки
**Команды:**
```bash
# Ручной запуск
bash scripts/backup-db.sh

# Проверить файл
ls -la ~/backups/db/
```
**Файлы:** Добавить в README.md секцию про бэкапы
**Проверка:** Бэкап создан, документация обновлена
**Результат:** Задача завершена ✅
```

**Пример 2: Контентный план (видео)**

```markdown
# Видео "5 автоматизаций для фрилансера" - План реализации

**Цель:** Выпустить видео с 5 практическими кейсами
**Срок:** 3 дня
**Зависимости:** Доступ к примерам автоматизаций, OBS, микрофон

---

### Шаг 1: Исследование и структура (2 часа)

**Что делаем:** Выбрать 5 кейсов, написать структуру
**Действия:**
- Проверить аналитику: какие темы заходят (skills/analytics)
- Посмотреть конкурентов: что уже снято
- Выбрать 5 кейсов с конкретными результатами
**Результат:** Документ со структурой:
```
1. Intro (30 сек) - hook + обещание
2. Кейс 1: [название] - 3 мин
3. Кейс 2: [название] - 3 мин
4. Кейс 3: [название] - 3 мин
5. Кейс 4: [название] - 3 мин
6. Кейс 5: [название] - 3 мин
7. Outro (1 мин) - CTA
```

---

### Шаг 2: Подготовка демо (3 часа)

**Что делаем:** Настроить каждую автоматизацию для записи
**Действия:**
- Открыть каждый workflow
- Подготовить тестовые данные
- Проверить что всё работает
**Проверка:** Каждый кейс можно показать live
**Результат:** 5 готовых демо

---

### Шаг 3: Запись (4 часа)

**Что делаем:** Записать screencast + talking head
**Действия:**
- Настроить OBS (1920x1080, микрофон)
- Записать intro/outro (talking head)
- Записать 5 screencast демо
- Записать voiceover если нужно
**Проверка:** Все клипы записаны, звук чистый
**Результат:** Сырые файлы готовы к монтажу

---

### Шаг 4: Монтаж (5 часов)

**Что делаем:** Собрать финальное видео
**Действия:**
- Склеить клипы по структуре
- Добавить переходы и графику
- Добавить музыку (royalty-free)
- Цветокоррекция
- Экспорт (1080p, H.264)
**Проверка:** Просмотреть целиком, проверить звук
**Результат:** Финальный файл video.mp4

---

### Шаг 5: SEO и публикация (1 час)

**Что делаем:** Оптимизировать и выложить
**Действия:**
- Передать в skills/youtube-seo для генерации метаданных
- Создать превью (thumbnail)
- Загрузить на YouTube
- Написать пост-анонс в Telegram (skills/copywriter)
**Проверка:** Видео опубликовано, анонс отправлен
**Результат:** Контент запущен ✅
```

**Пример 3: Организационный план**

```markdown
# Продление документов - План реализации

**Цель:** Продлить визу и страховку до конца года
**Срок:** 2 недели
**Зависимости:** Паспорт, текущие документы, деньги на счету

---

### Шаг 1: Собрать документы (1 день)

**Что делаем:** Подготовить пакет документов
**Документы:**
- [ ] Паспорт (оригинал + копия)
- [ ] Текущая виза
- [ ] Справка с банка (за 3 месяца)
- [ ] Договор аренды
- [ ] Фото 3x4 (2 шт)
**Проверка:** Все документы в папке, копии сделаны
**Результат:** Пакет готов

---

### Шаг 2: Записаться на приём (1 день)

**Что делаем:** Забронировать слот онлайн
**Сайт:** [ссылка на сервис записи]
**Контакт:** +XXX-XXX-XXXX (горячая линия)
**Проверка:** Получено подтверждение на email
**Результат:** Дата приёма: [дата]

---

### Шаг 3: Визит в офис (полдня)

**Что делаем:** Подать документы лично
**Адрес:** [адрес]
**Время:** [время приёма]
**Взять с собой:**
- Папку с документами
- Деньги на пошлину ($XX)
- Паспорт
**Проверка:** Получена квитанция о приёме
**Результат:** Заявка подана

---

### Шаг 4: Получить документы (через N дней)

**Что делаем:** Забрать готовые документы
**Когда:** Позвонить через [N] дней узнать статус
**Проверка:** Документы на руках, проверить даты
**Результат:** Задача завершена ✅
```

## 🔧 Настройка под себя

### Уровень детализации

По умолчанию: атомарные шаги (2-10 мин).

Можете запросить:
```
"Дай общий план без деталей" → высокоуровневые этапы
"Максимально детально" → каждое действие отдельно
```

### Формат сохранения

По умолчанию: Markdown файл в `memory/plans/`.

Альтернативы:
- Notion (если интегрирован)
- GitHub Issue
- Todoist/OmniFocus
- Прямо в чат (для простых планов)

### Интеграция с другими скиллами

**С `brainstorming`:**
```
brainstorming выбирает подход → writing-plans создаёт 
детальный план реализации выбранного варианта
```

**С `business-architect`:**
```
business-architect валидирует идею → writing-plans пишет 
план MVP с конкретными шагами
```

**Для контента:**
```
writing-plans создаёт общий план → 
copywriter/youtube-seo выполняют свои этапы
```

### Автоматическое выполнение

После создания плана можно:
```
"Запусти субагента на выполнение" → автономное выполнение
"Добавь в крон на завтра 9:00" → отложенный запуск
```
