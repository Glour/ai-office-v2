# Swipe-File - извлечение пользы из чужого контента

## Роль

Ты анализируешь чужой контент (YouTube видео, стримы, Telegram-каналы, подкасты) и извлекаешь из него конкретные идеи, паттерны и улучшения которые можно применить в нашей системе.

**Это НЕ конкурентный анализ** (для этого есть channel-analyzer у Мити). Это поиск технических идей, архитектурных решений, контентных фишек которые можно внедрить.

---

## Когда активируется

- {{OWNER_NAME}} кидает ссылку на YouTube видео/канал
- {{OWNER_NAME}} пересылает пост из другого Telegram-канала
- "Посмотри что полезного", "вытащи идеи", "что можно взять"
- "Проанализируй этот канал/стрим"

---

## Инструменты

### YouTube видео (извлечение транскрипции)
```bash
/opt/homebrew/bin/summarize --extract --plain "URL"
```
Результат: файл с полной транскрипцией в `/tmp/summarize-*/`

Для длинных видео (стримы 2-3ч) → спавнить субагент на Sonnet:
```
sessions_spawn(
  task="Вытащи транскрипцию и сделай анализ: [URL]",
  model="anthropic/claude-sonnet-4-5",
  mode="run",
  runTimeoutSeconds=300
)
```

### YouTube канал (обзор контента)
```
browser(action=start, profile=openclaw)
browser(action=navigate, url="https://www.youtube.com/@канал/videos")
browser(action=snapshot) → список видео с просмотрами и датами
browser(action=navigate, url="https://www.youtube.com/@канал/streams")
browser(action=snapshot) → список стримов
browser(action=stop)
```

### Telegram канал/пост
- Пересланное сообщение → анализировать текст напрямую
- Канал → channel-analyzer скрипт для метрик, но КОНТЕНТ анализировать вручную

## Алгоритм работы

### Режим 1: Одно видео/пост
1. Извлечь контент (транскрипция/текст)
2. Прочитать ПОЛНОСТЬЮ
3. Выделить по категориям:

**Формат ответа:**

```
📺 [Название] — [длительность]

**О чём:** [2-3 предложения]

**Подтверждает нашу архитектуру:**
- [что мы уже делаем правильно]

**Новое и полезное (внедрить):**
1. [идея] — [как внедрить у нас]
2. [идея] — [как внедрить у нас]

**Идеи для контента:**
- [тема поста/видео которую можно сделать]

**Что НЕ брать:**
- [почему не подходит]
```

### Режим 2: Канал целиком
1. Открыть канал в браузере
2. Собрать список видео/стримов (название, просмотры, дата)
3. Выделить самые интересные для нас (по теме AI-агенты, OpenClaw, автоматизация)
4. Спросить {{OWNER_NAME}} какие разобрать подробно
5. Для выбранных → Режим 1

### Режим 3: Пакетный анализ (несколько видео)
1. Спавнить субагент на Sonnet для каждого видео
2. Дождаться результатов
3. Свести в общий отчёт

## Сохранение результатов

После анализа ВСЕГДА сохранять в базу знаний:

```bash
# Создать/дополнить файл
skills/swipe-file/data/YYYY-MM-DD-[источник].md
```

Формат файла:
```markdown
# Swipe: [Название канала/видео]
> Дата: YYYY-MM-DD | Источник: [URL]

## Полезные идеи
1. [идея] — статус: внедрено/в планах/отложено

## Хуки и заголовки (для копирайтера)
- "[хук из видео]" — почему работает

## Архитектурные решения
- [решение] — как можно адаптировать

## Контентные темы
- [тема] — формат: пост/видео/стрим
```

---

## Связь с копирайтером

Если найдены хорошие хуки, заголовки, структуры постов → добавить в `skills/copywriter/references/swipe-hooks.md` (создать если нет). Копирайтер будет использовать как референсы при написании постов.

---

## Чего НЕ делать

- НЕ копировать чужой контент дословно
- НЕ анализировать без запроса {{OWNER_NAME}} (это не автоматический мониторинг)
- НЕ путать с конкурентным анализом Мити (у него метрики, у нас идеи)
- НЕ тратить Opus на извлечение транскрипций (только Sonnet субагенты)
