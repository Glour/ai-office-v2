# Product Validator 🔍

Автоматическая проверка продуктов Team Lead перед передачей на ревью.

## Когда использовать

- Шаг 4 пайплайна (безопасность) - вместо ручного grep
- Шаг 5 пайплайна (качество) - дополнение к ручной проверке
- Тройная проверка (проход 1: безопасность) - автоматизация

## Запуск

```bash
# Автоопределение типа
python3 skills/product-validator/scripts/validate.py <путь>

# Явный тип
python3 skills/product-validator/scripts/validate.py <путь> --type skill
python3 skills/product-validator/scripts/validate.py <путь> --type material
python3 skills/product-validator/scripts/validate.py <путь> --type github

# С автоисправлением тире
python3 skills/product-validator/scripts/validate.py <путь> --fix
```

## Что проверяет

### Для всех типов
1. **Безопасность** - личные данные (имена, ID, адреса, телефоны, ники), токены/ключи API
2. **Пути** - абсолютные пути `/Users/...`, `/home/...`
3. **Стиль** - длинное тире (заменить на дефис), канцелярит

### Для скиллов (+)
4. YAML frontmatter (name, description, triggers)
5. Размер SKILL.md (< 500 строк)
6. References упомянуты и существуют
7. Есть примеры использования

### Для материалов группы (+)
4. Dual format (MD + PDF + AGENT.md)
5. Обязательные разделы: Что это, Зачем, Как настроить, Проверка, Ошибки, Итого

### Для GitHub (+)
4. README.md, LICENSE, INSTALL.md, .gitignore

## Результат

```
==================================================
РЕЗУЛЬТАТ: ПРОЙДЕНО / НАЙДЕНЫ ПРОБЛЕМЫ
  Проверок: N
  ✅ Пройдено: X
  ⚠️ Предупреждений: Y
  ❌ Ошибок: Z
==================================================
```

Exit code: 0 = чисто, 1 = есть ошибки.

## Алгоритм работы

1. Запусти `validate.py` на папке продукта
2. Если ошибки → исправь → запусти повторно
3. Если только предупреждения → оцени, нужно ли исправлять
4. Если чисто → переходи к следующему шагу пайплайна

**Важно:** валидатор НЕ заменяет ручную проверку логики и "глазами подписчика". Он автоматизирует рутинные проверки (безопасность, стиль, структура).
