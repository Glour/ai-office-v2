# Researcher 🔍

Универсальный ресёрчер. Один скилл — доступ ко всем инструментам поиска. Находит что угодно и где угодно.

---

## Когда использовать

- "Что это за проект?" — разбор ссылки/репо/продукта
- "Что нового у конкурентов?" — мониторинг каналов
- "Найди информацию про X" — глубокий ресёрч по теме
- "Что обсуждают в чате?" — анализ Telegram групп
- "Тренды за последний месяц" — Reddit + X + Web
- "Посмотри это видео, что полезного" — анализ контента
- "Что пишут про OpenClaw?" — мониторинг упоминаний

---

## Алгоритм: определи тип задачи → выбери инструмент

### 1. Быстрый поиск (факт, определение, "что это")
```
web_search → web_fetch (если нужны детали) → ответ
```
**Инструменты:** `web_search`, `web_fetch`
**Время:** 30 секунд

### 2. Разбор ссылки (URL, GitHub репо, статья, продукт)
```
web_fetch (markdown) → анализ → ответ с выводами
```
**Инструменты:** `web_fetch`, `Lightpanda` (если web_fetch не справился)
**Время:** 1-2 минуты
**Lightpanda:**
```bash
scripts/lightpanda fetch --dump markdown <url>
scripts/lightpanda fetch --dump semantic_tree_text <url>  # если нужна структура
```

### 3. Глубокий ресёрч по теме (отчёт с источниками)
```
Скилл deep-research-pro → мульти-источниковый отчёт
```
**Инструменты:** скилл `deep-research-pro` (внутри использует web_search + web_fetch + синтез)
**Время:** 5-15 минут
**Когда:** "разберись в теме X", "напиши отчёт про Y", "сравни A и B"

### 4. Тренды и обсуждения (что говорят за 30 дней)
```
Скилл last30days → Reddit + X + Web → дайджест
```
**Инструменты:** скилл `last30days`
**Время:** 5-10 минут
**Когда:** "что обсуждают про X", "тренды в AI за месяц", "что нового в нише"

### 5. Telegram каналы и чаты (посты, реакции, боли)
```
channel-analyzer → анализ постов → выводы
```
**Инструменты:** скилл `channel-analyzer` + Telethon юзербот
**Команда:**
```bash
python3 skills/channel-analyzer/scripts/analyze-channel.py --channel "<username или id>" --days N --no-comments
```
**Время:** 2-5 минут
**Когда:** "что пишут в канале X", "проанализируй конкурента", "что в чате подписчиков"
**⚠️ Telethon сессия может быть залочена если другой процесс использует. Подожди или убей процесс.**

### 6. Конкуренты (мониторинг каналов)
```
channel-analyzer по списку → сводный отчёт
```
**Данные:** `data/competitors.md` (если есть в workspace агента)
**Когда:** "что нового у конкурентов", "еженедельный обзор"

### 7. YouTube видео (транскрипт + анализ)
```
Скилл tubescribe → транскрипт → анализ
```
**Инструменты:** скилл `tubescribe`
**Когда:** "посмотри это видео", "что там полезного", "вытащи идеи"

### 8. Чужой контент (swipe file — идеи и паттерны)
```
Скилл swipe-file → что взять для себя
```
**Инструменты:** скилл `swipe-file`
**Когда:** "проанализируй этот канал", "что можно взять", "вытащи идеи из поста"

### 9. Reddit (треды, обсуждения)
```
Скилл reddit → поиск + чтение тредов
```
**Инструменты:** скилл `reddit` + `skills/reddit/scripts/reddit.mjs`
**Когда:** "что на Reddit про X", "найди обсуждение Y"

### 10. RSS и блоги (мониторинг обновлений)
```
Скилл blogwatcher → отслеживание новых постов
```
**Инструменты:** скилл `blogwatcher`
**Когда:** "следи за блогом X", "есть ли новые посты"

### 11. Статистика каналов (аналитика YouTube + Telegram)
```
Скилл analytics → цифры, ERR, просмотры, топ постов
```
**Инструменты:** скилл `analytics`
**Скрипты:**
```bash
python3 skills/analytics/scripts/tg-stats.py --period=7d     # Telegram
python3 skills/analytics/scripts/yt-deep-stats.py --days=30   # YouTube
```
**Когда:** "статистика канала", "как идут просмотры", "ERR за неделю"

## Каскад (если первый инструмент не справился)

```
web_search → не нашёл?
  → web_fetch (прямой URL) → не рендерится?
    → Lightpanda (быстрый парсинг) → не отдаёт JS?
      → agent-browser (полный Chromium) → скриншот + snapshot
```

SearxNG как fallback если Brave API не работает:
```bash
curl "http://localhost:8888/search?q=ЗАПРОС&format=json"
```

## Формат результата

Короткий запрос → короткий ответ (3-5 предложений).
Глубокий ресёрч → структурированный отчёт:

```
## [Тема]

### Ключевые находки
1. [Факт + источник]
2. [Факт + источник]
3. [Факт + источник]

### Выводы
[Что это значит для нас]

### Источники
- [URL 1]
- [URL 2]
```

---

## Чего НЕ делать

- НЕ выдумывать если не нашёл — честно сказать "не нашёл"
- НЕ давать устаревшие данные — проверять дату источника
- НЕ парсить платный контент (paywall) — сказать что закрыто
- НЕ запускать Telethon если сессия залочена — подождать
